Sliver-C2 framework#

Nhận chall với 2 file DMP và pcap check signature thì file DMP là file minidump

Question 1 ta sử dụng Windbg để debug file minidump chương trình đã bị crash tại process AMUSED_MILKSHAKE tiến hành dump process đó ra kiểm tra với

đây là thông tin của module dùng lệnh .writemem ‘%USER PROFILE%\outputfolder…’ start end

Khi có file ta đưa vào virustotal để quick scan ta có thể thấy đây là một trojan sliver search google sliver framework

Đáp án 1 : sliver

Question2: Check file pcap ta thấy được các gói http nghi vấn đến từ 192.169.1.108 chứa các chuỗi hex, base64, words lạ Đáp án 2 : 192.168.1.108

Question3: sau khi đã biết được attacker sử dụng Sliver-C2 framework tìm các bài blog có liên quan https://www.immersivelabs.com/blog/detecting-and-decrypting-sliver-c2-a-threat-hunters-guide

trong blog có nói rõ về cách extract session key từ memory dump session key dùng để decrypt các payload trên network traffic Không may mắn là ta phải tìm key theo cách thủ công với Hex và thu nhỏ phạm vi tìm kiếm với Pattern

với các thông tin này ta viết script chuyển file DMP => sang dạng hex

Lọc pattern

output

Đáp án 3 : 11f4e5b7e21870c8f44143464b1b80b7b03f9852d782ef6d90b81f626f291401

Question 4: Sau khi có được session key ta tiến hành xử lý payload trong file pcap với tool được đề cập trong blog

sử dụng silver-pcap-parse.py để lấy payload ra do không có dấu hiệu payload ở DNS nên ta chỉ filter cho http

xong sử dụng sliver_decrypt.py cùng với key session để decrypt file http_message.json

như có vẻ không hoạt động tốt sau vài giờ kiểm tra tôi biết vấn đề nằm ở các định dạng file http-message.json

ta thấy các định dạng đã được đưa về hex không đúng như định dạng trong script decrypt ở chỗ words tôi bôi đỏ đúng định dạng phải như này và gzip-b64 tôi sẽ đưa về b64 để dễ cho script decrypt hoạt động tốt tránh lỗi trong việc xử lý gunzip lúc này tôi sẽ sử dụng file http-message.json làm tương tự với các process trong file http-message.json sau khi đã sửa file http-message ta decrypt được

ta có được payload

sau xong ta lấy tất cả hex của message data đem đi decode probobuf https://protobuf-decoder.netlify.app/

file payload chưa được decode protobuf bây giờ vào link trên để decode khi decode ta thấy đây là một cuộc tấn công ransomware folder dc nhắm tới C:\Users\Administrator\Downloads
